Referência de nós

O catálogo de todos os nós disponíveis para montar árvores. As condições e ações vêm do mesmo registry que o editor usa para montar a paleta e validar — então as descrições aqui refletem exatamente o que o jogo e o simulador executam.

ℹ Nomes legíveis & grupos

No editor e no simulador, cada condição/ação aparece com um nome legível em português (ex.: Skill em área (AoE), HP do dono (abaixo)) e a paleta é organizada em grupos (Vida/SP, Ameaça, Alvo, Ataque, Movimento, Skills, Buffs/cura, Dono, Eleanor & Sera). O código original (ex.: UseAoESkill) — que esta tabela usa na coluna name — continua sendo o identificador salvo na árvore e aparece na dica (tooltip) ao passar o mouse no nó ou na opção.

Status

Todo nó retorna um destes a cada avaliação:

  • SUCCESS — cumpriu o objetivo neste tick.
  • FAILURE — não se aplica ou não conseguiu agir.
  • RUNNING — ação em andamento; continua no próximo tick.

Composites

Aceitam vários filhos. São reativos (sem memória): a árvore reavalia tudo a cada tick.

NóCamposSemântica
selectorchildren, label?OU de prioridade. Tica os filhos em ordem; vence o primeiro com SUCCESS ou RUNNING; falha só se todos falharem. A ordem dos filhos é a prioridade.
sequencechildren, label?E. Tica em ordem; para no primeiro FAILURE ou RUNNING; sucesso só se todos passarem.
parallelchildren, policy, label?Tica todos os filhos. policy="all" → sucesso se todos; policy="any" → sucesso se ao menos um.

Decorators

Aceitam um filho.

NóCamposSemântica
inverterchildTroca SUCCESS↔FAILURE; RUNNING passa.
succeederchildForça SUCCESS (exceto quando RUNNING). Torna um ramo opcional.
cooldownms, childBloqueia (FAILURE) por ms milissegundos após o filho ter SUCCESS. Mantém estado.
limitermax, child, key?Permite no máximo max sucessos do filho; depois falha.

Condição (check)

O nó Condição (check) é um condicional com um filho opcional.

NóCamposSemântica
checkname, params?, child?Avalia a condição name. Se verdadeira e há child, executa-o e devolve o status dele; se verdadeira e não há child, devolve SUCCESS. Se falsa, devolve FAILURE sem executar o filho.

Na prática, check é como a árvore amarra “se condição X, então faça Y” num único nó (ex.: HasOwnerCommand → HandleOwnerCommand).

monsterCheck — condicional por monstro-alvo

monsterCheck executa seu único filho dependendo de qual monstro é o alvo atual. Ele olha a classe do monstro alvo e a compara com um monstro e/ou grupo cadastrados (veja Monstros & skills).

NóCamposSemântica
monsterCheckmonster (id, 0=nenhum), group (id, 0=nenhum), negate (bool), childCasa quando o alvo é o monstro monster ou está no grupo group. Com negate, casa quando não é e não está. Se casa e há child, executa-o; sem child, devolve SUCCESS; se não casa, FAILURE. Sem alvo atual, sempre FAILURE.

Folhas: condições

Uma condição (dentro de uma Condição/check) referencia uma destas pelo name. As que têm parâmetros recebem-nos em params.

CondiçãoParâmetrosDescrição
HasOwnerCommand—Há comando do dono pendente neste tick.
HpBelowpctHP% do homúnculo abaixo de pct.
HpAbovepctHP% do homúnculo ≥ pct.
SpAbovepctSP% do homúnculo ≥ pct.
BeingAttacked—Algum monstro mira no homúnculo.
OwnerUnderAttackcount?O dono está sendo atacado por pelo menos count monstros (padrão 1).
HasValidTarget—Existe alvo válido (em vista, vivo, alcançável).
InAttackRange—Alvo atual dentro do alcance de ataque normal.
TooFarFromOwnerdistDistância do dono acima do limite.
CanEngage—Condições gerais para buscar alvo (HP/SP, não SuperPassive, não standby).
ShouldFlee—HP abaixo de FleeHP e sob ataque.
OwnerHpBelowpctHP% do dono abaixo de pct (ex.: curar o dono).
OwnerHpAbovepctHP% do dono ≥ pct.
MobbedcountPelo menos count monstros atacando o homúnculo (marca berserk).
StyleIsstyle (power|grapple)O estilo atual (Eleanor) é o pedido.

TargetIsBoss—O monstro alvo é Boss/MVP. Útil p/ proibir skills em boss ou trocar de tática.

Skills automáticas (dirigidas pelo perfil do homúnculo)

Estas ações resolvem qual skill usar a partir do perfil do tipo de homúnculo — você não indica IDs.

AçãoDescrição
UseMainSkillUsa a 1ª skill single-target utilizável da lista de prioridade do papel (0..N skills; lista vazia ⇒ não age). Gates: UseHomunSSkillAttack (em alcance) / UseHomunSSkillChase (aproximando). Knobs aceitam override por nó (param do nó > Config global).
UseAoESkillUsa a 1ª AoE utilizável da lista de prioridade (0..N); lista vazia ⇒ não age. Sem mainAtk (ex.: Dieter), a AoE é a ofensiva principal e dispara com ≥1 alvo; com mainAtk exige AutoMobCount alvos (economia de SP: single no isolado, AoE no aglomerado). AutoMobMode=0 desliga; AoEFixedLevel fixa o nível; AoEMaximizeTargets mira no aglomerado.
UseOffensiveBuffMantém TODOS os buffs ofensivos da lista (0..N; recasta cada um ao expirar — ex.: Bayeri usa Golden Ferse e Angriff Modus; vazia ⇒ nenhum).
UseDefensiveBuffMantém TODOS os buffs defensivos da lista (0..N; Amistr Bulwark, Granitic Armor, Stein Wand, ...; vazia ⇒ nenhum). Nível por skill via tela de Skills.
UseHealSelfCura a si quando HP% < HealSelfHP (Chaotic Blessing).
UseHealOwnerCura o dono quando HP% < HealOwnerHP (Healing Hands/Silent Breeze/Chaotic).
UseOwnerBuffMantém buff no dono (Painkiller da Sera).
UseSummonInvoca minions contra o alvo (Summon Legion). Nível/política: param do nó > padrão da tela de Skills (homun_summons.json) > perfil.
UseCastlingCastling para tirar mobs do dono quando ele está cercado (Amistr).

Skills explícitas (você escolhe a skill)

Use estas para comportamentos customizados apontando uma skill específica. No editor, o campo skill é preenchido pelo seletor de skills por categoria.

AçãoParâmetrosDescrição
UseSkillskill, level, on (enemy|owner)Usa uma skill de dano em alvo único ou área. on define o centro das skills de área.
UseSkillBuffskill, levelUsa um buff em si mesmo (não recasta enquanto durar).
SetStylestyle (power|grapple)Garante o estilo (Eleanor): conjura Style Change se necessário.
UseCombocombo, windowExecuta um combo no alvo (sequência de golpes).
UseEleanorOffensestyle, comboSpheres, window, ...Eleanor: combo + estilo + esferas + segurança do Agarrão num só nó. Precedência: param do nó > padrão da tela de Skills (homun_skills.json) > config. Veja Monstros & skills.

Como os nós de skill se adaptam por homúnculo

A mesma árvore atende os 9 tipos de homúnculo. As ações automáticas resolvem a skill concreta pelo perfil do tipo, obtido do homúnculo (e, para Homunculus S, do tipo base). Por exemplo, UseMainSkill vira Caprice no Vanilmirth, Moonlight no Filir, Stahl Horn no Bayeri; UseHealOwner só faz algo no Lif; UseCastling só no Amistr. Por isso você monta a árvore uma vez e ela serve a todos os tipos — basta trocar o homúnculo no contexto. No editor, o rótulo de cada ação automática mostra as skills efetivas (nome + nível) do homúnculo do contexto, e o inspetor traz um link “⚙ Parâmetros desta skill” para o modal de parâmetros globais da ação (AutoMobCount, AttackRange, HealSelfHP, etc.) — valem para todos os homúnculos. O override por nó fica recolhido em “⚙ Avançado”. A precedência é param do nó > override do homúnculo > parâmetros globais > Config global > padrão. Para sobrepor os parâmetros num homúnculo específico, marque “sobrepor parâmetros” naquele papel no modal Skills por homúnculo (booleanos são sim/não).

Exemplo de árvore

Trecho da árvore padrão mostrando check, selector, sequence e folhas:

{
  "type": "selector",
  "label": "root",
  "children": [
    { "type": "check", "name": "HasOwnerCommand", "label": "comando",
      "child": { "type": "action", "name": "HandleOwnerCommand" } },
    { "type": "check", "name": "ShouldFlee", "label": "sobrevivencia",
      "child": { "type": "action", "name": "Flee" } },
    { "type": "sequence", "label": "Engajar", "children": [
        { "type": "selector", "label": "Definir alvo", "children": [
            { "type": "check", "name": "OwnerUnderAttack",
              "child": { "type": "action", "name": "AcquireOwnerAttacker" } },
            { "type": "check", "name": "HasValidTarget" },
            { "type": "check", "name": "CanEngage",
              "child": { "type": "action", "name": "AcquireTarget" } } ] },
        { "type": "selector", "label": "combate-acao", "children": [
            { "type": "action", "name": "UseAoESkill" },
            { "type": "action", "name": "UseMainSkill" },
            { "type": "check", "name": "InAttackRange",
              "child": { "type": "action", "name": "AttackTarget" } },
            { "type": "action", "name": "ChaseTarget" } ] } ] }
  ]
}
